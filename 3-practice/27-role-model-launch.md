## Глава 27. Запуск ролевой модели: 6 шагов

### Шаг 1. Покер-функций

Участники поочерёдно называют действия, необходимые кругу для достижения результата. Можно пропустить ход. За два часа обычно формируется список из ~500 действий. Каждое отвечает на вопрос **«Что делать?»**

### Шаг 2. Объединить действия в роли

Объединить по смыслу. Обычно получается 10–15 ролей. С этой задачей неплохо справляется ChatGPT. Не бойтесь детализировать.

### Шаг 3. Установить ожидаемый результат

Для каждой роли — чёткий измеримый результат, востребованный другими ролями. Если результатом никто не пользуется — роль избыточна.

### Шаг 4. Определить необходимые навыки

Ключевые навыки для достижения результатов. Специалист сразу понимает, что ему нужно.

### Шаг 5. Спланировать ресурсы

Определить FTE для каждой роли (может быть < 1). Распределить роли между сотрудниками по предпочтениям.

### Шаг 6. Выбрать лидера

Команда выбирает лидера путём простого голосования.

**Важно:** всё созданное — лишь первая версия. Роли, ресурсы и навыки будут меняться по мере решения ограничений.

> **Что говорит наука.** Лоулер (1994) показал, что компетентностная модель повышает гибкость и развитие сотрудников. Процесс «покер-функций» — коллективное определение действий, необходимых команде — отражает принцип партиципативного дизайна работы, который Хакмен (*Leading Teams*, 2002) считал ключевым для командной эффективности. А практика выбора лидера командой подтверждена исследованиями Лидена и соавторов (2008, *The Leadership Quarterly*): лидерство-служение, включая фасилитацию выбора, предсказывает приверженность и результативность сильнее, чем назначение сверху.
