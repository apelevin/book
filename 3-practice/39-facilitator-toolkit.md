## Глава 39. Инструментарий фасилитатора

Правократия — не только философия и теория. Это набор конкретных инструментов, которые мы используем каждый день. Ниже — практические методы работы с командой, культурой и ограничениями, сгруппированные по задачам.

### Инструменты наблюдения

#### «Муха на стене»

Придите на встречу команды как молчаливый наблюдатель. Не участвуйте, не комментируйте. Фиксируйте:
- Кто говорит, а кто молчит?
- Какие темы избегаются?
- Кто к кому обращается?
- Есть ли моменты ложного согласия?

Записывайте конкретные поведения, а не интерпретации. Этот метод помогает лидеру-фасилитатору увидеть реальную групповую динамику. Особенно полезен для обнаружения речевых паттернов «не-Правократии»: «жду, когда скажут», «это не моя зона», «лучше не выносить на обсуждение».

**Когда использовать:** перед governance-встречей, при оценке психологической безопасности, при подозрении на скрытую иерархию.

#### Теневое наблюдение

Договоритесь сопровождать коллегу в течение рабочего дня. Наблюдайте молча: действия, инструменты, прерывания, среду. Минимизируйте разговоры — все вопросы оставьте на дебриф после наблюдения.

Метод раскрывает то, что люди сами не замечают: привычные неэффективности, скрытые ограничения, нормализованную боль.

**Когда использовать:** лидер круга хочет понять реальную работу команды; подготовка к делению круга; улучшение взаимодействия между кругами.

#### Карта социальных связей

Визуализируйте реальные связи в команде или между кругами:

1. Определите участников
2. Зафиксируйте: кто с кем взаимодействует, как часто, по каким вопросам
3. Нанесите на карту: тип связи (формальная/неформальная), сила, направление
4. Анализируйте: ключевые коннекторы, изолированные люди, узкие места

В плоской структуре нет формальной иерархии, но есть неформальные центры притяжения. Если все потоки идут через одного человека — это скрытая иерархия, которую нужно распределить.

**Когда использовать:** при делении круга (где естественные линии разделения?), для диагностики коммуникационного налога, для поиска скрытых лидеров.

### Инструменты диагностики культуры

#### Словесные ассоциации

Быстрый метод для измерения «температуры» культуры:

1. Подготовьте список ключевых слов: «лидер», «ответственность», «свобода», «ошибка», «контроль», «круг», «роль», «конфликт»
2. Произносите по одному — участники моментально отвечают первым, что приходит в голову
3. Фиксируйте все ответы без оценки
4. Анализируйте паттерны: какие слова вызывают страх? Энтузиазм? Замешательство?

**Когда использовать:** до и после трансформации (как меняется отношение к слову «руководитель»?), при диагностике нового круга, при онбординге как тест на культурный fit.

#### «Нарисуй опыт»

Вместо слов — рисунок. Рисование обходит вербальные фильтры и выявляет истинные эмоции:

1. Раздайте бумагу и маркеры
2. Задайте вопрос: «Как ты чувствуешь себя в команде?» или «Нарисуй свой рабочий день»
3. Подчеркните: навык рисования не важен, стик-фигуры — отлично
4. Каждый объясняет свой рисунок: что значат символы, размеры, расположение

Удивительно, сколько скрытого напряжения или радости проявляется через простые рисунки.

**Когда использовать:** на ретроспективах («нарисуй наш квартал»), при онбординге («нарисуй первую неделю»), как альтернативный формат peer-review.

#### Когнитивные карты

Каждый рисует на чистом листе: «Как устроена наша компания?» Без подсказок и образцов. Затем все объясняют свои карты.

Сравните: где совпадения? Где разрывы? Если карта содержит «верх» и «низ» — плоская структура ещё не прижилась в головах. Если круги нарисованы изолированно — нужно работать над связями.

**Когда использовать:** раз в полгода для отслеживания трансформации, при «раздвоении бизнес-личности» (видят ли люди старую или новую компанию?).

### Инструменты работы с ограничениями

#### Анализ ошибок

Системный подход к превращению ошибок в улучшения:

1. Соберите данные об ошибках: обращения в поддержку, жалобы, провалившиеся гипотезы, инциденты
2. Классифицируйте:
   - **Оговорки** (slips) — человек знал, как правильно, но ошибся
   - **Заблуждения** (mistakes) — человек действовал по неверной модели
   - **Обходные пути** (workarounds) — система не работает, и люди нашли свой путь
3. Для каждого паттерна: что в системе привело к ошибке? (не «кто виноват»)
4. Приоритизируйте: частота x влияние = приоритет исправления

Обходные пути — самый ценный источник. Они показывают, где система не работает, и одновременно содержат зерно решения.

**Когда использовать:** на ретро-встречах, при формировании паспорта ограничения, при анализе провалившихся гипотез.

#### Анализ потоков

Визуализация движения информации, решений или клиентских запросов через систему:

1. Определите, что отслеживаете: проблемы, решения, клиентские запросы
2. Зафиксируйте каждый шаг: переходы, ожидания, передачи между кругами, точки принятия решений
3. Визуализируйте как диаграмму
4. Ищите: узкие места (где застревают вопросы?), петли (куда возвращаются проблемы?), тупики (где теряется информация?)

**Когда использовать:** для поиска скрытой иерархии (все потоки через одного человека?), при делении круга (как перераспределить потоки?), для оптимизации взаимодействия между кругами.

#### Анализ активностей

Разбиение сложной деятельности на составные действия — расширенная версия «покера функций»:

1. Определите деятельность (например, «работа продукт-менеджера»)
2. Разбейте на все дискретные действия — записывайте каждое отдельно
3. Для каждого: что делается, какие инструменты, сколько времени, какой результат, кто ещё вовлечён
4. Выявите: дублирование, узкие места, действия без результата

**Когда использовать:** при запуске ролевой модели (~500 действий → 10-15 ролей), при ежеквартальном пересмотре ролей, при обнаружении перегрузки конкретной роли.

### Инструменты генерации и проверки гипотез

#### Сценарии

Гипотеза — это абстракция. Сценарий превращает её в конкретную историю:

1. Выберите персонажа — реального человека или типичного представителя
2. Напишите историю: что он делает, думает и чувствует, когда гипотеза внедрена
3. Включите контекст: среду, триггеры, неожиданные ситуации
4. Обсудите командой: реалистично? Где ломается?

Пишите сценарии для разных типов: «А если новичок? А если 10 человек? А если кризис?»

**Когда использовать:** для каждой гипотезы OKR, перед масштабированием успешных экспериментов, в формате HADI (гипотеза → действие-сценарий → метрики → вывод).

#### «Заголовки будущего года»

Мощное упражнение для визионирования:

1. Каждый пишет 3-5 заголовков, которые хочет увидеть о компании через год
2. Заголовки должны быть конкретными: «ПравоТех сократил время подготовки документов на 40%», а не «Компания стала лучше»
3. Поделитесь, обсудите: общие темы? Неожиданные амбиции?
4. Лучшие заголовки становятся ориентирами для OKR

Напишите и **негативные заголовки**: «Что напишут, если мы провалимся?» Это обнажает риски, которые команда предпочитает не замечать.

**Когда использовать:** при планировании OKR (начните с заголовков, затем перейдите к ограничениям), на стратегических сессиях визионерского круга, при запуске нового продукта.

#### Быстрое прототипирование процессов

Гипотезу нужно проверить, а не обсуждать месяц:

1. Возьмите гипотезу
2. За 30-90 минут создайте простейшую версию: на бумаге, в таблице, в мессенджере, в виде ручного процесса
3. Цель — «думать через делание», а не создать идеальное решение
4. Покажите команде → соберите обратную связь → итерируйте

В Правократии прототипируют не только продукты, но и процессы: «Как будет работать новый формат встречи? Давайте попробуем прямо сейчас.» И ролевые модели: «Давайте попробуем эту роль неделю и посмотрим.» И финансовые схемы: «Как будет выглядеть распределение бонусов по новой формуле? Посчитаем на примере.»

#### Ролевая игра

Проживание ситуации через разыгрывание ролей:

1. Определите сценарий и роли: клиент, лидер, новичок, критик
2. Каждый получает цели и ограничения своего персонажа
3. Разыграйте ситуацию с импровизацией
4. Обсудите: что было естественным? Что неловким? Какие возможности открылись?

**Когда использовать:**
- Репетиция сложных разговоров: как дать обратную связь? Как сказать «нет» идее CEO?
- Тренировка консент-процесса: один играет «возражающего», остальные интегрируют
- Подготовка к онбордингу: как объяснить Правократию новичку?
- Проигрывание конфликтов: «пересесть на одну сторону стола» становится буквальным

### Инструменты эмпатии

#### «Побудь в чужой роли»

Проживание ограничений другого человека:

- **«Побудь новичком»** — отключите все привилегии на день: только базовый доступ, никаких инсайдерских знаний. Как чувствует себя человек в первую неделю?
- **«Побудь клиентом»** — пройдите полный путь: от первого контакта до решения проблемы. Не пропускайте шагов, не используйте знания «изнутри».
- **«Побудь в другой роли»** — проведите день в роли, которую обычно не выполняете. Разработчик — день в поддержке. Продажник — день в разработке.

Эти упражнения разрушают слепые пятна и создают настоящую эмпатию — не на уровне «я понимаю», а на уровне «я прожил».

#### «Экскурсия по кругу»

Попросите коллегу провести «экскурсию» по своему рабочему пространству — физическому или цифровому:

1. Он показывает и объясняет: что для чего, что важно, что мешает
2. Обращайте внимание на то, что выделяет, пропускает, чем гордится

**Варианты:**
- **Между кругами:** лидер показывает, как устроена работа его команды — для других кругов
- **Обратная экскурсия:** новичок проводит экскурсию для команды через месяц — что он видит, чего вы не замечаете?
- **Цифровая экскурсия:** «Покажи свой рабочий экран» — какие инструменты, какие вкладки открыты, где трение?

#### «Думай вслух»

Попросите коллегу выполнить рабочую задачу, непрерывно комментируя свои мысли:

- «О чём ты думаешь сейчас?»
- «Почему ты это сделал?»
- «Что тебя остановило?»

Записывайте моменты замешательства, раздражения, удивления. Метод показывает скрытые ограничения в процессах, которые «вроде бы работают». Особенно эффективен при тестировании новых инструментов — включая AI-помощников.

#### Культурный дневник

Раздайте участникам набор для самонаблюдения на 1-2 недели:

- Дневник с ежедневными вопросами: «Что принесло удовольствие?», «Где было трение?», «Какое решение не удалось принять самостоятельно?»
- Карточки для фото: «Сфотографируй момент командной работы», «Сфотографируй то, что мешает»

Через две недели соберите и проанализируйте. Метод даёт доступ к повседневной реальности, которую невозможно увидеть на встречах.

**Когда использовать:** между большими опросами вовлечённости, для глубокого понимания жизни конкретного круга.

### Инструменты стратегии

#### Профили архетипов

Создайте 3-4 исследовательских портрета людей в компании:

1. На основе реальных данных выделите архетипы: «Ветеран-скептик», «Амбициозный миллениал», «Зумер-по-умолчанию», «Тихий профессионал»
2. Для каждого: имя, бэкграунд, мотивация, страхи, типичный день
3. При принятии решений спрашивайте: «Как это воспримет Скептик? А Зумер?»

Обновляйте ежегодно. Профили меняются вместе с культурой.

#### Долгосрочное прогнозирование

Структура для работы визионерского круга:

1. Определите ключевые тренды: технологии (AI), демография (зумеры), экономика, регулирование (правотех)
2. Для каждого тренда: что будет через 5 лет? А через 10?
3. Скомбинируйте в сценарии будущего
4. Для каждого сценария: какие возможности? Какие угрозы? Что мы должны начать делать уже сейчас?

#### Анализ траектории

Соберите историю трансформации компании:

1. Ключевые вехи: решения, события, метрики, потери, победы
2. Создайте таймлайн
3. Ищите повторяющиеся паттерны и точки перелома
4. Проецируйте: куда ведёт траектория? Что мы упускаем?

Документация истории нужна и для обучения новичков, и для стратегических решений, и просто для того, чтобы помнить: путь, который мы прошли, — не случайность.
